# OPIc Topic Folder Workspace

Treat the current directory as the project root.

This workspace is for building memorization-friendly OPIc study materials for a Korean IT professional preparing for OPIc difficulty 5-6.

## Core persona defaults
- Works in IT / software engineering
- Main work includes legacy system migration, system improvement, and AI-agent-related tasks
- Lives in an apartment or shared housing with family
- Prefers spoken English that is natural, stable, and realistic for an intermediate learner
- Needs reusable story assets across multiple question types

## Default output policy
The default deliverable is **topic folders containing one markdown file per question**, not one long inline dump.

Always save outputs under:
- `outputs/opic/YYYY-MM-DD/`

Create these companion files when generating a full pack:
- `00-index.md`
- `00-persona-sheet.md`
- `00-story-asset-bank.md`
- one folder per topic
- one markdown file per question inside each topic folder
- `90-study-plan.md` (optional but recommended)

## Required question file rule
Every question markdown file must include all of the following sections:
1. 암기 포인트 및 흐름
2. 한국어 전문
3. 영어 전문
4. 키 문장 및 단어
5. 키 한국어 문장 및 대응되는 영어표현 여러개

You may add extra helpful sections, for example:
- Follow-up Hooks
- Shadowing Lines
- Risky Expressions → Easier Alternatives
- Reusable Story Asset Tags
- Quick Answer Version

## Formatting rule
- Use markdown headings consistently.
- Add blank lines between sections.
- Split long paragraphs into short readable blocks.
- Prefer 2-4 short sentences per paragraph block.
- For Korean and English full scripts, keep line breaks natural for speaking practice.
- For expression banks, use bullets and spacing generously.
- Never produce dense wall-of-text question files.

## Writing rules
- Keep persona facts consistent across topics.
- Reuse a compact set of story assets instead of inventing a new life story for every question.
- Prefer concrete details, simple emotions, and time flow.
- Avoid essay-like tone and over-advanced vocabulary.
- One main idea per sentence whenever possible.
- Keep answers flexible enough for follow-up questions.

## Review rules
Before finishing, check:
- Did every requested topic become its own folder?
- Does each folder contain one markdown file per question?
- Does every question file contain the 5 mandatory sections?
- Are Korean and English versions aligned in meaning?
- Does each key Korean sentence have multiple English variants?
- Are line breaks clean and readable?
